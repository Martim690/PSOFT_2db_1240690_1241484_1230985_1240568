# US206 - Calculate Total Operational Hours per Aircraft

## 1. User Story
> As an ATCC, I want to calculate the total operational hours for each aircraft in my fleet.

## 2. Acceptance Criteria
- The request must trigger the generation of an operational hours report for the entire fleet.
- The system must aggregate data from all physical aircraft instances.
- The response must map each aircraft's `registrationNumber` to its `totalFlightHours`.
- On success, the system must return HTTP `200 OK` with the collection of operational hours payloads and RESTful HATEOAS links.

## 3. Pre-conditions
- The actor must be authenticated and authorized with either the `ATCC` or `ADMIN` role.
- Aircraft instances should exist in the database to generate a meaningful report.

## 4. Post-conditions
- The state of the database remains completely unchanged (read-only operation).
- The ATCC is provided with the required operational data for fleet tracking and maintenance scheduling.

## 5. Main Success Scenario
1. The actor sends a `GET /api/aircrafts/operational-hours` request to generate the report.
2. The `AircraftController` handles the request and delegates the processing to the `CalculateAircraftOperationalHoursUseCase`.
3. The Use Case retrieves the full list of `Aircraft` entities from the `AircraftRepository`.
4. The system iterates over the list, extracting the `RegistrationNumber` (as a text string) and the `totalFlightHours` from each entity.
5. The system encapsulates these specific fields into an `AircraftOperationalHoursResponse` DTO for each aircraft.
6. The system returns an HTTP `200 OK` response containing the mapped list and HATEOAS links (including a link to view each specific aircraft's details).

## 6. Alternative / Exception Flows
| Step | Condition | System Response |
|------|-----------|-----------------|
| 1 | The user is not authenticated or lacks the required roles | HTTP `401 Unauthorized` or `403 Forbidden` |
| 3 | There are no aircraft instances currently registered in the system | HTTP `200 OK` with an empty collection |

## 7. Design Justification
- **DTO Projection:** The system utilizes `AircraftOperationalHoursResponse` to ensure that only the strictly necessary fields (registration number and hours) are sent to the client, saving bandwidth and hiding internal domain complexity.
- **HATEOAS Navigation:** By including an `aircraft-details` link inside each response item, the frontend can easily navigate from this high-level report to the deep dive of a specific aircraft without hardcoding URL structures.
