# US214 - List Active Routes Ordered by Operational Criteria

## 1. User Story
> As a Backoffice Operator, I want to view a list of all active routes ordered by specific criteria (e.g., alphabetically or by distance) to facilitate flight planning lookups.

## 2. Acceptance Criteria
- The request parameter can accept sorting variables (e.g., `sortBy=distance`, `order=desc`).
- The payload must omit all routes marked as inactive (`active = false`).
- On success, returns HTTP `200 OK` with the ordered array.

## 3. Pre-conditions
- The user must be authenticated with standard system access tokens.

## 4. Post-conditions
- Read-only lookup; database structural mapping is unaffected.

## 5. Main Success Scenario
1. The actor sends a `GET /api/routes/active?sortBy=distance` request.
2. The `RouteController` forwards query sort settings down.
3. The use case contacts the repository to retrieve the targeted sorted stream data subset.
4. Records are collected and served back in an HTTP `200 OK` response schema layout.

## 6. Alternative / Exception Flows
| Step | Condition | System Response |
|------|-----------|-----------------|
| 1 | Unauthorized role execution | HTTP `401 Unauthorized` |

## 7. Design Justification
- **Database Sorting Level:** Delegation of filtering and sorting parameters to the database query layer via JPQL indices ensures that the Spring memory heap size stays small and optimized.
