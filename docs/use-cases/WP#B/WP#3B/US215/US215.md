# US215 - Calculate Total Operational Network Distance Analytics

## 1. User Story
> As a Backoffice Operator, I want to retrieve the total collective unique distance of the entire active route network map to evaluate global fleet infrastructure coverage.

## 2. Acceptance Criteria
- The system aggregates the `distance` value of all currently active routes.
- Inactive routes must be completely ignored from the analytics pipeline.
- On success, returns HTTP `200 OK` with a single structured numeric payload response value representing the net sum.

## 3. Pre-conditions
- The user must be authenticated into the system environment.

## 4. Post-conditions
- Read-only numerical analytics summation evaluation.

## 5. Main Success Scenario
1. The actor requests a network health query via `GET /api/routes/analytics/total-distance`.
2. The application layer interceptor matches the path logic.
3. The query extracts a computed SQL sum summary calculation mapping from the `RouteRepository`.
4. The calculation outcome payload wraps inside an object context and returns an HTTP `200 OK`.

## 6. Alternative / Exception Flows
| Step | Condition | System Response |
|------|-----------|-----------------|
| 1 | Invalid JWT bearer authority configuration context | HTTP `401 Unauthorized` |

## 7. Design Justification
- **SQL Aggregation Utilization:** Utilizing an operational query aggregate function (`SUM(r.distance)`) instead of mapping entire lists into memory structures reduces processor overhead.
