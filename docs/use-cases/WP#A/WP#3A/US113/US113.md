# US113 - Find Flight Routes by Airport

## 1. User Story
> As a Backoffice Operator, I want to query and list all flight routes associated with a specific airport (either as origin or destination) to evaluate regional coverage.

## 2. Acceptance Criteria
- The request must provide a valid 3-letter airport code (e.g., `LIS`) as a path or query variable.
- The response must filter the entire dataset, returning only records where `originIataCode == target` or `destinationIataCode == target`.
- On success, returns HTTP `200 OK` with an array collection. If no routes are found, returns an empty array with HTTP `200 OK`.

## 3. Pre-conditions
- The actor must be authenticated with an appropriate organizational role.

## 4. Post-conditions
- No state changes occur (Read-Only query operation).

## 5. Main Success Scenario
1. The actor sends a `GET /api/routes/airport/{iataCode}` request.
2. The `RouteController` handles the target criteria parameters.
3. The `FindRoutesByAirportUseCase` calls the specialized query method on the `RouteRepository`.
4. The database filters records matching the IATA identifier.
5. The system constructs a collection of DTO records and issues an HTTP `200 OK` response.

## 6. Alternative / Exception Flows
| Step | Condition | System Response |
|------|-----------|-----------------|
| 1 | Invalid authorization context | HTTP `401 Unauthorized` |
| 2 | The input parameter does not match syntactic IATA formats (e.g., length != 3) | HTTP `400 Bad Request` |

## 7. Design Justification
- **Custom Repository Queries:** This leverage optimized JPQL indexing/queries (e.g., `findByOriginIataCodeOrDestinationIataCode`) to ensure fast response extraction without pulling all database records into memory.
